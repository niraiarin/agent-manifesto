# Disagreement-Tier Active-Learning Runbook

Companion to issue #677. Describes how to compute 3-way label disagreement
tiers and route only the high-uncertainty subset to human annotation,
saving roughly 60% of labeling effort relative to a full-corpus review.

## Tiers

| Tier | Definition | Annotator action |
|------|------------|------------------|
| **0** | All 3 models agree | Auto-accept majority label as GT, no human time |
| **1** | 2 models agree, 1 dissents | Quick review (~30s each), use majority as default with minority shown as "reconsider" hint |
| **2** | All 3 disagree (each picks a different label) | Deep review (~2-3 min each), annotator decides from scratch with all 3 model labels visible |

Estimated yields on 500 candidates (per #677):
- tier 0: ~100–150 (auto-accept)
- tier 1: ~200–250 (quick review)
- tier 2: ~100–150 (deep review)
- Total human time: ~8 hours vs ~20 hours for full review.

## Inputs

Three label JSONL files, all keyed by the same `id` field, one record per prompt:

| Source | Field for label | Generated by |
|--------|----------------|--------------|
| Qwen 35B-A3B | `label` | `qwen_labels.py` |
| Qwen 27B-Q4 | `label` | `qwen_labels.py` (different model checkpoint) |
| mDeBERTa | `predicted_label` | `serve_encoder.py` /classify endpoint or batch run |

Optional fields preserved: `prompt`, `session_id`, `predicted_confidence`,
`predicted_probs`. The first model that has a `prompt` field provides the
prompt text to the output record.

## Step 1 — Compute tiers

```bash
python3 docs/research/routellm-phase3/classifier/compute_disagreement_tiers.py \
  --qwen35b   docs/research/routellm-phase3/label-data/qwen-labels-500.jsonl \
  --qwen27b   docs/research/routellm-phase3/label-data/qwen-labels-27b-q4.jsonl \
  --mdeberta  docs/research/routellm-phase3/label-data/mdeberta-predictions-500.jsonl \
  --output    docs/research/routellm-phase3/label-data/disagreement-tiers.jsonl \
  --summary
```

`--summary` prints a stderr breakdown:

```
tier distribution:
  tier 0 (all-agree    ):   142 ( 28.4%)
  tier 1 (2-vs-1       ):   213 ( 42.6%)
  tier 2 (all-disagree ):   145 ( 29.0%)
  total                    :   500
```

`--allow-missing` records `null` for absent models instead of dropping the
prompt; tier classification still works on the partial data (degraded
confidence).

Output record shape:

```jsonc
{
  "id": "gt-qwen-0001",
  "prompt": "/verify the recent commit",
  "tier": 1,
  "model_labels": {
    "qwen35b":  "cloud_required",
    "qwen27b":  "cloud_required",
    "mdeberta": "hybrid"
  },
  "majority_label":   "cloud_required",
  "minority_models":  ["mdeberta"],
  "agreement_count":  2
}
```

## Step 2 — Split by tier and dispatch to annotators

Use `jq` to slice by tier; feed each slice through `annotator_kit.py`.

```bash
TIERS_FILE=docs/research/routellm-phase3/label-data/disagreement-tiers.jsonl
KIT_DIR=docs/research/routellm-phase3/label-data/human-gt-by-tier

# tier 0 → write directly to human-gt-majority.jsonl (no human action needed)
jq -c 'select(.tier == 0) | {id, prompt, gt_label: .majority_label, source: "auto-accept-tier0"}' \
  "$TIERS_FILE" > docs/research/routellm-phase3/label-data/human-gt-tier0-auto.jsonl

# tier 1 → quick-review packet (annotator confirms or overrides majority)
jq -c 'select(.tier == 1)' "$TIERS_FILE" > /tmp/tier1.jsonl
python3 docs/research/routellm-phase3/classifier/annotator_kit.py \
  --candidates /tmp/tier1.jsonl \
  --annotators alice bob \
  --output-dir "$KIT_DIR/tier1" \
  --taxonomy-ref ../analysis/label-guide.md

# tier 2 → deep-review packet (annotator decides from scratch)
jq -c 'select(.tier == 2)' "$TIERS_FILE" > /tmp/tier2.jsonl
python3 docs/research/routellm-phase3/classifier/annotator_kit.py \
  --candidates /tmp/tier2.jsonl \
  --annotators alice bob \
  --output-dir "$KIT_DIR/tier2" \
  --taxonomy-ref ../analysis/label-guide.md
```

The annotator markdown packet renders `model_labels` and `minority_models`
as part of the prompt context, so the human can see how each model voted
before deciding.

## Step 3 — Aggregate human GT

After both annotators return their JSONLs:

```bash
# Inter-annotator agreement on tier 1+2 (use existing kappa.py from #674)
python3 docs/research/routellm-phase3/classifier/kappa.py \
  --a "$KIT_DIR/tier1/alice.jsonl" \
  --b "$KIT_DIR/tier1/bob.jsonl" \
  --report docs/research/routellm-phase3/analysis/annotator-agreement-v2.md

# Build the merged GT (auto-accept tier 0 + majority of tier 1+2)
# implement merger in a follow-up script: build_human_gt.py
```

## Gate (per #677)

| Metric | PASS | CONDITIONAL | FAIL |
|--------|------|-------------|------|
| Inter-annotator Cohen's kappa (tier 1+2) | ≥ 0.75 | 0.60–0.75 | < 0.60 |
| Tier 0 auto-accept rate | ≥ 25% | 15–25% | < 15% |
| mDeBERTa retrained ECE | ≤ 0.10 | 0.10–0.15 | > 0.15 |

PASS gates the publication of `human-gt-majority.jsonl` as the new GT
reference. CONDITIONAL routes a third arbiter for the disagreements
between alice and bob. FAIL means the underlying taxonomy itself needs
revision before further annotation.

## Output schema reference

`disagreement-tiers.jsonl` is JSON-Lines. Each record has:

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | Prompt identifier (matches input) |
| `prompt` | string | First non-empty `prompt` / `text` from the input models |
| `tier` | 0 \| 1 \| 2 | See table above |
| `model_labels` | object | `{qwen35b, qwen27b, mdeberta}`; values are valid label strings or null |
| `majority_label` | string \| null | Defined for tier 0 and tier 1 only |
| `minority_models` | array \| null | Defined for tier 1 (list of dissenter model keys); also populated for degraded tier 2 |
| `agreement_count` | int | 3 (tier 0), 2 (tier 1), 1 or 0 (tier 2) |

Compatible with `additionalProperties` in any downstream schema; readers
should tolerate unknown fields for forward compatibility with annotator
notes added by `annotator_kit.py`.
