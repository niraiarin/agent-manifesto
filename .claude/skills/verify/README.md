# verify

P2（Cognitive Separation of Concerns）の実装。Worker の成果物を独立したコンテキストで検証する。

コード変更、設計文書、設定変更のレビューに使う。
コミット前、PR 前、または品質に不安がある時に呼び出す。
リスクレベルに応じて検証手段を選択する。

## 起動トリガー

`/verify` または「レビューして」「検証して」「チェックして」

## リスクレベルと検証手段

| リスク | 検証手段 | 条件 |
|--------|---------|------|
| Low | Subagent | コンテキスト分離のみで十分 |
| Medium | Subagent + 人間確認 | フレーミング独立性が必要 |
| High | 人間レビュー | 評価者独立性が必要 |
| Critical | 全4条件 | 外部レビュー必須 |

## 4条件 (VerificationIndependence)

1. コンテキスト分離 — Worker の思考過程が Verifier に漏洩しない
2. フレーミング独立性 — Worker の問題設定に Verifier が拘束されない
3. 評価者独立性 — 同一モデルファミリーでない
4. 実行自動性 — 検証が自動的にトリガーされる
