# spec-driven-workflow

仕様駆動・テスト駆動開発の完全ワークフロー。

条件付き公理系（Lean 設計書）の生成から、テスト計画導出、実装、
閉環トレーサビリティ検証、保守運用までを一貫して実行する。
既存スキル群を統合した開発ライフサイクルの司令塔。

## 起動トリガー

`/spec-driven-workflow` または「仕様駆動」「spec-driven」「テスト駆動」「TDD」「ワークフロー」「開発ライフサイクル」

## 思想

```
設計書 = T₀ ∪ Γ\T₀(requirements, constraints) ⊢ D_project
```

設計書は自然言語文書ではなく条件付き公理系（Lean 文書）。
全ての主張は条件付き公理系の命題に帰着可能でなければならない（閉環トレーサビリティ）。

## ワークフロー

```
条件付き公理系 ─[validates]─→ テスト ─[verifies]─→ 実装
       ↑                                              │
       └──────────────[justifies]──────────────────────┘
```
